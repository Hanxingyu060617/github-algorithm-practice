# C语言 HelloWorld 增强版
在基础 HelloWorld 输出的基础上，实现安全的自定义输入函数 `input()`，解决原生输入处理的常见问题（换行符残留、超长输入异常、缓冲区乱码等）。

## 核心功能
 基础功能：打印 `HelloWorld!`
核心增强：自定义 `input()` 函数，安全读取用户输入
  自动去除输入末尾换行符
  处理输入异常与超长输入场景
  清空缓冲区残留数据，避免乱码

## 环境要求
编译器：支持C99及以上标准（GCC/Clang/MSVC）
系统：Windows/Linux/macOS（跨平台兼容）

注意：比起上一版本
 使用 static 缓冲区的优势：
   避免野指针：静态存储区内存生命周期与程序一致，返回指针始终有效；
   简化调用：无需手动定义缓冲区/释放内存，适配入门场景；
   高效复用：仅初始化一次，减少内存开销。
 不使用 static 的优势（进阶参考）：
   无覆盖问题：多份输入内容可独立保留，支持复杂输入场景；
   内存可控：按需分配/释放，无全局生命周期的资源浪费；
   线程安全：避免多线程下的内存竞态问题，可扩展性更强；
   灵活适配：支持自定义缓冲区大小，适配不同输入长度。

##ESP32环境配置和代码测试
本次实验主要是进行Arduino IDE的安装和给ESP32进行环境配置，并测试代码，观察灯的亮灭情况和频率
注意：
1.如果串口监视器波特频不一致，串口监视器会显示乱码，没有输出正常的文本信息
原因是：串口通信属于异步串行通信，通信双方（开发板与串口监视器）必须使用完全相同的波特率才能正确解析数据
2.修改delay(1000)的作用是让程序暂停1000毫秒，数值减小，闪烁频率变快，增大变慢。
